APIs REST con PHP

Objetivos de Aprendizaje
  • Comprender los principios de las APIs REST
  • Entender los métodos HTTP y su relación con CRUD
  • Conocer el formato JSON para intercambio de datos
  • Aprender sobre los códigos de estado HTTP
  • Implementar buenas prácticas en diseño de APIs REST

¿Qué es REST?

REST (Representational State Transfer) es un estilo de arquitectura para diseñar servicios web. Fue presentado por Roy Fielding en su tesis doctoral en 2000 y se ha convertido en el estándar de facto para el diseño de APIs web modernas.

Principios fundamentales de REST
  • Interfaz uniforme: Los recursos se identifican mediante URIs y se manipulan mediante representaciones.
  • Sin estado: Cada petición contiene toda la información necesaria, no depende de peticiones anteriores.
  • Cacheable: Las respuestas pueden ser cacheable para mejorar el rendimiento.
  • Cliente-servidor: Existe una separación clara entre cliente y servidor.
  • Sistema en capas: La arquitectura puede estar compuesta por múltiples capas jerárquicas.

Recursos y URLs

En una API REST, todo es un recurso. Un recurso puede ser un objeto, un servicio, o cualquier entidad que pueda ser identificada, nombrada y manipulada a través de la web. Cada recurso es accesible a través de una URL única.

Recurso URL Descripción
Colección /usuarios Representa a todos los usuarios
Elemento /usuarios/123 Representa al usuario con ID 123
Subrecurso /usuarios/123/pedidos Todos los pedidos del usuario 123
Filtrado /usuarios?activo=true Usuarios que están activos

Métodos HTTP y operaciones CRUD

Los métodos HTTP definen las acciones que se pueden realizar sobre los recursos. Estos se corresponden con las operaciones CRUD (Create, Read, Update, Delete) de las bases de datos.

métodos_http.md 

| Método HTTP | Operación CRUD | Descripción                                 | Ejemplo                                   |
|-------------|----------------|---------------------------------------------|-------------------------------------------|
| GET         | Read           | Obtener datos                               | GET /usuarios - Listar todos los usuarios |
| GET         | Read           | Obtener un recurso específico               | GET /usuarios/1 - Obtener usuario con ID 1|
| POST        | Create         | Crear un nuevo recurso                      | POST /usuarios - Crear un nuevo usuario   |
| PUT         | Update         | Actualizar un recurso existente (completo)  | PUT /usuarios/1 - Actualizar usuario 1    |
| PATCH       | Update         | Actualizar parcialmente un recurso          | PATCH /usuarios/1 - Actualizar una parte  |
| DELETE      | Delete         | Eliminar un recurso                         | DELETE /usuarios/1 - Eliminar usuario 1   |

Formato JSON

JSON (JavaScript Object Notation) es el formato estándar para intercambiar datos en APIs REST. Es ligero, fácil de leer para humanos y fácil de procesar para las máquinas.

ejemplo_usuario.json 

{
  "id": 1,
  "nombre": "Juan Pérez",
  "correo": "juan@example.com",
  "activo": true,
  "rol": "admin",
  "creado": "2025-01-15T10:30:00Z"
}

Códigos de estado HTTP

Los códigos de estado HTTP comunican el resultado de una petición. Es importante utilizarlos correctamente en tu API REST.

Código Descripción Uso en APIs
200 OK La petición se completó con éxito Respuesta a GET, PUT, PATCH exitosos
201 Created Recurso creado con éxito Respuesta a POST exitoso
204 No Content Éxito pero sin contenido en la respuesta Respuesta a DELETE exitoso
400 Bad Request La petición tiene algún error Datos de entrada inválidos
401 Unauthorized Se requiere autenticación El cliente no está autenticado
403 Forbidden No tiene permisos Usuario autenticado sin permisos
404 Not Found Recurso no encontrado ID o ruta no existe
500 Internal Server Error Error del servidor Errores de programación o base de datos

Buenas prácticas para diseñar APIs REST

  1. Usa sustantivos, no verbos en las URLs (ej. /usuarios en lugar de /getUsuarios)
  2. Versiona tu API (ej. /api/v1/usuarios)
  3. Utiliza correctamente los códigos de estado HTTP
  4. Implementa paginación para colecciones grandes
  5. Permite filtrado, ordenación y búsqueda mediante parámetros de consulta
  6. Documenta tu API para que otros la puedan usar fácilmente
  7. Implementa validación de datos tanto en cliente como en servidor
  8. Maneja correctamente los errores con mensajes descriptivos
  9. Asegura tu API con HTTPS y estrategias de autenticación apropiadas
En resumen

Las APIs REST siguen principios de diseño que las hacen escalables, mantenibles y fáciles de usar. Utilizan URLs para identificar recursos, métodos HTTP para manipularlos, y códigos de estado para comunicar resultados. En las próximas lecciones, veremos cómo implementar y consumir APIs REST con PHP.
